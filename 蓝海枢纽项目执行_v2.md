## 一、总纲提示词（每次对话开头使用）

```
你是我团队的深远海能源系统研究助理，协助完成国家能源集团"揭榜挂帅·未来能源产业赛道"参赛项目
《蓝海枢纽：面向深远海能源岛的源储算用协同调度与多形态价值转化路径研究》。

【项目定位】
"源(多源供电)-储(构网型混合储能)-算(海底绿色算力)-用(多形态输出)"四位一体深远海能源岛方案，
核心是回答：如何通过源储算用协同调度把深远海清洁能源价值最大化转化为电力/绿氢绿氨/算力服务/
海洋用能；这套方案相比单一外送模式在经济性、消纳率、可靠性、环保性上能提升多少，如何落地。

【系统架构】
多源能源供给层（漂浮式风电为主85%+、漂浮式光伏互补、波浪潮流能预留）→
构网型混合储能中枢（短时构网储能负责秒级-分钟级调频调压/黑启动 + 长时储能/制氢负责
小时级-季节级转移与价值升级）→ 价值转化层（海底绿色算力集群PUE<1.1作为柔性负荷 +
海水制氢转化单元）→ 多形态输出应用层（电力外送/绿氢绿氨船运/算力服务/海洋综合用能），
由智能协同调度大脑统一优化分配。

【核心痛点】海缆造价随离岸距离指数级上升（超100km后超1500万元/公里，损耗超8%）；
深远海弃风率普遍超10%；离网系统缺乏电压频率支撑；单一卖电模式回收期过长。

【模型细节的处理原则】能源岛模型的整体框架内容与项目计划书一致，但具体公式、参数、
约束条件需要通过检索学术论文获取，不能自行编造核心公式。凡涉及公式推导，请优先基于
你检索到的、来源明确的文献给出，并标注文献来源类型；无法查证来源的公式或参数，
必须明确标注"[需查证文献支撑]"。

【角色扮演】每次对话请先确认我这次需要你扮演"产业调研员/系统架构工程师/优化建模师/
仿真分析员/产业落地顾问/资料检索员/图表可视化顾问"中的哪一个，再按对应任务卡输出。
所有数据必须标注来源类型（政府官网/行业协会报告/上市公司公告/核心期刊论文/企业访谈
一手数据），无法查证的一律标"[假设值，待企业调研校准]"，不得编造具体数字冒充真实数据。
```

## 三、对比方案的确定方式（对应反馈第3点）

不再预设"5种固定命名方案"，而是在Node1系统架构草图和模型基础框架都出来之后（约Day3-4），
由模型组和系统架构组共同讨论，基于"电力外送/绿氢绿氨/算力服务"三种输出的不同比例组合，
确定若干组对比方案，建议至少覆盖：
- 单一外送为主（低储能/低转化占比，作为基准对照）
- 均衡分配型（三种输出比例相对平均）
- 某一价值形态优先型（例如算力优先或制氢优先，视哪种在经济性测算里更具潜力而定）

提示词：
```
角色：系统架构工程师 + 优化建模师协同。
基于当前的系统架构参数（各设备容量、储能规模上限）和模型框架，帮我设计3组具有明显
区分度的输出比例分配方案，用于第五章的对比仿真。每组方案请说明：
（1）三种输出（电力/氢氨/算力）的大致分配逻辑；
（2）为什么这组比例值得作为对比对象（即它验证了什么假设）；
（3）预计在哪个指标上会有最明显的差异。
```

## 五、多目标权重的确定方法（对应反馈第6点）

检索市面上常见的做法作为参考基准：

```
角色：优化建模师。
请检索能源系统多目标优化相关文献，总结目前学术界和行业里常用的、用于给"经济性/
环保性/可靠性"（或类似的多维目标）分配权重的方法，至少覆盖：
1. 层次分析法（AHP）——适用场景、如何构造判断矩阵
2. 熵权法——适用场景、优缺点
3. TOPSIS等综合评价法与权重法的配合使用方式
4. 不需要主观定权重的方法：如帕累托前沿（NSGA-II等多目标遗传算法），说明它和
   加权法相比的优劣
请结合我们的场景（深远海能源岛的电力/氢能/算力协同调度）说明哪种方法更适合我们，
并给出2-3篇可供参考的相关文献方向（不需要编造具体文献，说明去哪类期刊/数据库找）。
```

## 系统架构组与模型组的对接

第三章回答"有哪些设备、参数范围多大、怎么协同"，第四章要把这些参数变成决策变量的 取值边界和约束条件，第五章要用具体数值定义对比方案和仿真输入——三者共用同一套数字。 不建议两组各自闷头写自己的章节，而是建一份两组共用、独立于两章叙述文字之外的 对接文档来同步。

###  参数对接表（系统架构组交付，模型组消费）（以实际为准）

| 参数名称                   | 所属模块      | 取值/区间 | 单位      | 状态（草案/已定/企业校准） | 影响的模型部分         |
| -------------------------- | ------------- | --------- | --------- | -------------------------- | ---------------------- |
| 风机额定功率与装机占比     | 多源供给层    |           | MW / %    |                            | 决策变量上界           |
| 构网储能功率/容量/响应速度 | 储能中枢-短时 |           | MW·MWh/ms |                            | 电压频率约束、爬坡约束 |
| 制氢设备效率与调节范围     | 储能中枢-长时 |           | %/MW      |                            | 制氢功率决策变量边界   |
| 算力负荷功率区间与响应速度 | 价值转化层    |           | MW/s      |                            | 柔性负荷约束           |
| 海缆传输容量与损耗率       | 输出应用层    |           | MW/%      |                            | 外送功率约束           |

关键原则：

- 第三章TRL评估中标注为"技术尚不成熟/行业参数分散"的项，系统架构组要显式标记， 模型组据此把它列为第五章敏感性分析的变量，而不是当成固定常数写进约束条件。
- 装机容量类的数字要比两章的文字初稿更早锁定，因为第三节的3种对比方案设计依赖 这些容量上限——模型组没有这些数字就没法做方案分配。

###  一致性把关

组织架构里技术总监（统筹第三章撰写、同时负责模型约束审核）天然处于两组交界位置， 建议明确由她在V0→V1→V2每个版本节点签字确认"第三章里的数字"和"模型里用的数字" 是同一套，而不是等到Node5统稿时才发现对不上。

###  对接检查提示

```
角色：系统架构工程师 + 优化建模师协同评审。
这是系统架构组当前版本（V1/V2）的设备参数清单：[粘贴参数表]。
这是模型组当前使用的决策变量边界和约束条件：[粘贴模型片段]。
请检查两者是否一致，指出：
（1）模型里用到但架构参数表里没有覆盖的变量；
（2）架构参数表里给出但模型没有使用、可能被遗漏的约束；
（3）哪些参数属于TRL不成熟/取值分散的类型，建议放入第五章敏感性分析而非固定约束。
```

## 第一、二、六章的推进与修改要点

这三章不像第三/四/五章那样有强数值耦合，但有两个真正的风险点：**第一章和第二章 内容容易重复**，以及**第六章的经济性测算容易和第五章仿真结果脱节**。

### 10.1 第一章（绪论）与第二章（未来产业研判）的内容

两章不要各写各的痛点罗列，建议明确分工：

|            | 第一章 绪论                                                  | 第二章 未来产业研判                                        |
| ---------- | ------------------------------------------------------------ | ---------------------------------------------------------- |
| 核心任务   | 问题导向：用数据把痛点砸实，引出本项目解法的逻辑起点         | 定位导向：论证这个方向"是不是未来产业、现在是不是窗口期"   |
| 内容重点   | 研究背景、开发现状、核心挑战（输电成本/弃风率/离网稳定性/单一收益模式，量化数字为主） | 能源岛定义、国内外趋势对标、未来产业属性论证、优势与窗口期 |
| 不要做的事 | 不需要把第三章的具体技术方案展开，留轮廓即可                 | 不要重复罗列第一章已经讲过的开发痛点数据                   |

**第二章的官方依据（可直接引用，增强说服力）**：工信部等七部门2024年印发的 《关于推动未来产业创新发展的实施意见》中，"未来产业"被定义为由前沿技术驱动、 当前处于孕育萌发阶段或产业化初期、具有显著战略性/引领性/颠覆性/不确定性的 前瞻性新兴产业，并明确列出未来制造、未来信息、未来材料、**未来能源**、未来空间、 未来健康六大方向。深远海能源岛可直接对标"未来能源"方向；"十五五"规划纲要 还点名了氢能作为新的经济增长点，与本项目的绿氢/绿氨模块直接呼应。建议第二章 围绕这套官方定义和方向分类展开论证，而不是自造一套论证逻辑。
