# Blue Hub development specification

本仓库依据用户提供的《蓝海枢纽 Codex 模型开发总说明》开发。原说明定义了完整的七层架构、设备方程、经济与碳核算、多目标优化、动态可靠性、测试要求和 Phase 0–10 路线。本文件记录当前仓库必须执行的阶段门控。

## 共同原则

- 物理守恒、单位明确、参数可追溯、结果可复现；
- 逐时能量模型与秒级动态模型分离；
- 设备容量、运行调度、价值属性和碳属性分别记账；
- 不预设混合模式最优；
- 不用遗传算法替代透明的 LP/MILP 基线；
- 缺失值不得静默填零；
- 新参数必须具有单位、来源、来源等级和不确定性范围；
- 每个阶段必须保留可运行基线并更新测试与验证报告。

## 已完成：Phase 0

交付：仓库骨架、类型化 schema、确定性 24 h 数据、任意长度数据生成器、严格输入校验、单位检查、配置哈希和测试。

禁止：电池、制氢、算力、Pareto 和动态稳定优化逻辑。

## 已完成：Phase 1 / S0

只实现：

```text
wind availability -> critical load -> cable export -> cable loss -> curtailment -> revenue
```

验收：

- 24 h 与 8760 h 均可运行；
- 每小时海上功率平衡残差小于 `1e-6 MW`；
- 海缆不可用时外送为零；
- 无风时不能产生输出；
- 外送端与陆侧交付端电量及损耗分别记账；
- 负电价行为由显式合同/弃电规则决定；
- 年度收入和能量单调性通过独立手算与回归测试。

Phase 1 通过后，才依次加入电池、制氢、算力和集成优化。

## 已完成：Phase 2 / S1

在冻结S0回归结果的基础上加入构网型BESS的逐时能量代理模型，包括充放电功率、SOC、效率、自放电、年度周期边界、退化成本和应急储备。S1需通过无凭空增能、SOC边界、首尾状态、禁止同时充放电、储备裕度和零电池消融测试。

## 已完成：Phase 3 / S2

已加入淡化水物料成本—电解槽—储氢—销售链路。基础模型采用综合系统SEC，禁止再叠加已包含在SEC内的常规BOP能耗。每小时联合优化外送、电池充放、制氢、销售、弃电和关键负荷服务；氢库存和电池SOC均采用首尾闭合。

验收：

- 电解槽功率、制氢量、销售量、库存和水耗逐时守恒；
- 销售受外生需求上限与储罐上限约束，期末库存不得作为收入；
- 零制氢配置精确退化为S1；
- SEC、氢价、需求、储氢时长和海缆故障场景均可复现；
- 全年功率、SOC和氢库存残差受显式容差约束。

## 已完成：Phase 4 / S3

已加入绿色算力服务模型。算力主计量为MWh-IT，PUE将完成的IT工作转换为海上设施耗电；刚性任务存在显式SLA未完成量，柔性任务受队列容量、最大延迟和年度首尾状态约束。海底光缆被表示为可用率与服务输出能力共同约束的MWh-IT服务链路，未将光通信带宽错误映射为电力损耗。

验收：

- 无算力容量时精确退化到S0；
- PUE上升时，相同IT任务的设施用电上升；
- 刚性任务未完成量显式计入SLA成本；
- 柔性任务队列、最大延迟和年末队列均受审计；
- 光缆不可用时算力服务输出为零，恢复后仅柔性任务可补偿。

## 已完成：Phase 5 / S4

已将S1电池、S2制氢储氢和S3绿色算力放入单一逐时LP。电力外送、电池充放、制氢、氢销售、刚性算力、柔性算力、光缆服务交付和弃电共享同一个海上功率平衡，收入严格按电力、氢气和算力服务分别记账。

验收：

- 零算力S4严格退化到S2；零电池与制氢S4严格退化到S3；
- 电池SOC、氢库存、柔性队列均首尾闭合；
- 同一MWh不能重复形成电力、氢气或算力收益；
- 五种模式共享风资源、海缆、电价、氢价和算力合同的外生边界；
- 完成氢价—算力价、海缆故障、光缆故障和复合故障案例。

## 已完成：Phase 6 / S5

S5将大陆逐时电力接纳能力从海缆物理容量中分离出来，并加入可主动接入的全国弹性算力需求、燃料电池回发和跨季度储氢。电池、氢能和算力均可独立关闭，从而在相同的发电、电价、接纳能力与需求条件下比较八种资产组合。

验收：

- 逐时外送不得超过海缆可用容量与大陆接纳系数的乘积；
- 算力服务不得超过全国需求、IT容量和光缆服务能力，PUE换算后进入海上功率平衡；
- 燃料电池耗氢与发电量按低位热值效率换算，并与制氢、销售和库存共同守恒；
- 支持风电与海上光伏共同供能，零光伏配置保持原有风电结果；
- 完成电池、氢能和算力的全组合增量价值、交互项与Shapley分摊；
- 完成大陆接纳受限、72小时海缆故障、90日连续低风、等效海缆容量和海缆边际价值案例；
- 所有结果继续保持SOC、氢库存、算力队列、海上功率和陆侧损耗逐时闭合。

## 已完成：Phase 7 / S6

S6在固定风电、光伏和海缆公共边界下，将电池功率与能量、电解槽、储氢、燃料电池和算力—光纤容量改为连续投资决策。目标函数以年度运行边际扣除资本回收、固定运维和电解槽栈体更换成本后的净年值为准。S6规划容量可转换为固定配置，并由S5完整逐时模型回放。

验收：

- 电池功率成本与能量成本分别年化，所有灵活资产使用各自寿命和固定运维率；
- 零投资上界精确退化到S5直接输电，允许经济最优为零容量；
- 完成三档成本、三档大陆接纳约束、算力报价和氢价进入区间；
- 完成固定容量在8个样本外合成年份中的逐时回放；
- 完成90日低风下储氢—燃料电池保供容量规划，并区分商品收入与缺供避免价值；
- 所有容量和运行变量继续满足功率、SOC、氢库存、需求、光缆和海缆约束。

## 已完成：Phase 8 / S7

S7保留S6的逐时容量规划，在中国情境筛选中加入完整项目公共成本和可观测的登陆约束。风电、海上光伏和波浪能均可进入海上母线；波浪能暂以滞后平滑代理曲线进行互补性筛选。大陆接纳能力以登陆点绝对上限表达，不随海缆物理容量自动放大。输氢管道容量、距离相关投资和年化成本进入规划，氢气销售仍受需求、库存与管道共同约束。

验收：

- 在共同资源和完整公共成本边界下比较直接送电、单独海缆扩容、电网加固、综合能源岛和混合方案；
- 用2024年全国风电利用率构造全国参照校准，同时将局部登陆约束作为独立项目情景，不用全国均值替代登陆点数据；
- 完成算力服务价、氢价、可移动任务比例、低中高成本、风光波资源组合、72小时海缆故障和30日低风保供计算；
- 将商品收入、完整项目年化净值和关键负荷保供价值分开报告；
- 输出证据登记、文件哈希、代表性8760小时结果和48个固定随机时点审计；
- 保持外送、算力、管道、储电、储氢和海上/陆侧功率守恒约束，所有自动化测试继续通过。

## 已完成：Phase 9 / S8

S8将S7形成的固定容量向量放入六个统一的资源、登陆能力、电价、算力合同和氢价情景中逐时回放。容量在情景揭示前确定，运行调度在每个情景内优化。筛选同时报告期望完整项目年值、最差20%概率质量均值、风险调整年值和情景后悔值，并以独立压力案例检验登陆能力替代、合同门槛和低风保供。

验收：

- 直接送电及四个商业灵活方案使用同一组情景和权重，固定容量不得按情景事后重选；
- 最差概率质量允许截取边界情景的部分权重，权重归一、情景后悔值和候选完整性均受独立审计；
- 合同分析比较固定候选方案，不把每个价格点重新规划所得的完美容量误作可实现的合同响应；
- 登陆能力分析保持700 MW海缆物理容量不变，单独改变登陆点接纳上限；
- 低风可靠性分析将商品型制氢储氢与专用保供储备分开，并报告缺供电量、可避免缺供量和成本回收所需事件频率；
- 8760小时逐时结果保持海上和陆侧功率守恒，输出文件通过独立审计和全量回归测试。

## 下一阶段：Phase 10

以同址多年度风、光、波和海况资料替换筛选曲线，并接入拟登陆点的8760小时接纳能力、潮流约束、市场价格和扩网时序。算力侧需要分类型任务、可迁移比例、SLA和可融资客户合同；氢能侧需要用户负荷、交付质量、管径压力、压缩储存和全链条安全设计。容量规划将扩展为连续的两阶段随机投资和滚动调度，使用历史统计或经审议的联合分布估计情景概率，随后加入机组启停、可用率、检修、故障率、建设期融资、税费、债务、残值和碳属性，形成可用于项目尽调的NPV、IRR与容量建议区间。
