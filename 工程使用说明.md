# 蓝海枢纽子模型工程使用说明

## 工程作用

本工程用于建立“蓝海枢纽”项目中电力外送与海洋用能部分的可运行子模型。当前代码重点覆盖：

- 电力-海底电缆外送：计算海缆容量、登陆点接纳、电力损耗、电力收入和外送约束。
- 算力-光缆输出：计算数据中心设施用电、PUE 转换后的 IT 服务量、光缆服务能力约束和算力收入。
- 制氢-储氢-管道/船运输出：计算电解制氢量、储氢库存、管道/船运输出、损耗、氢气交付量和氢气收入。
- 海洋综合用能：计算岛上辅助负荷、海水淡化、海洋装备用能、缺供惩罚和服务量。
- 综合小时功率平衡：把电力外送、算力、制氢和海洋负荷接入同一个小时级调度检查框架。

当前版本是“可审计子模型原型”，不是完整优化求解器。它适合用于参数校准、典型日测算、约束检查和后续接入优化器前的公式验证。

## 参数配置

模型参数集中放在：

```text
submodules/configs/parameters.yaml
```

真实企业参数到手后，优先修改这个 YAML 文件，不要直接改公式代码。配置文件中已经给出每个参数的中文含义、单位和口径。

主要参数类别包括：

- `power_export`：海缆容量、登陆点接纳、海缆损耗、电价、外送变动成本、海缆投资和运维率。
- `compute`：算力设施容量、最小负荷、PUE、光缆服务能力、带宽、延迟、算力服务价格和变动成本。
- `hydrogen`：电解槽容量、制氢综合电耗、管道/船运能力、储氢容量、损耗率、氢价和运输成本。
- `marine`：岛上辅助负荷、淡化负荷、装备负荷、柔性比例、服务价值和缺供惩罚。

注意：当前只支持 YAML 配置文件，支持后缀为 `.yaml` 或 `.yml`。

## 环境要求

建议使用：

- Python 3.11 或更高版本
- uv 作为依赖和运行环境管理工具

如果新电脑没有安装 uv，可先执行：

```powershell
python -m pip install uv
```

## 运行测试

在项目根目录或任意位置打开 PowerShell，进入 `submodules` 目录：

```powershell
cd D:\project\submodules
$env:UV_CACHE_DIR='D:\project\.uv-cache'
uv run python -m unittest discover -s tests
```

如果输出类似下面内容，说明模型、参数读取和测试环境正常：

```text
Ran 17 tests
OK
```

## 运行示例

示例程序会自动读取：

```text
submodules/configs/parameters.yaml
```

运行命令：

```powershell
cd D:\project\submodules
$env:UV_CACHE_DIR='D:\project\.uv-cache'
uv run python examples\run_example.py
```

示例会生成 24 小时典型日的汇总结果，包括：

- 外送电量
- 实际送达电量
- 算力服务量
- 制氢量
- 氢气交付量
- 海洋负荷服务量
- 弃电量
- 总收入
- 总成本
- 经营毛利
- 功率平衡残差
- 约束违背次数

## 修改参数后的使用流程

推荐流程如下：

1. 打开 `submodules/configs/parameters.yaml`。
2. 把默认参数替换为企业提供的真实参数。
3. 运行测试，确认参数格式和范围没有问题。
4. 运行示例，查看 24 小时测算结果。
5. 如果需要年度结果，再接入 8760 小时出力、电价、需求和接纳能力时序。

## 当前模型边界

当前模型有意保持简化，主要用于跑通公式和接口：

- 不做详细电网潮流，只做海缆容量、登陆接纳能力和损耗约束。
- 不做通信网络路由，光缆只作为算力服务交付能力约束。
- 不做船舶逐航次调度，船运以等效 `kg/h` 能力表示。
- 不做完整财务模型，当前输出以运行收入、变动成本和经营毛利为主。
- 不做最优调度，当前示例使用简化贪心调度；如需最优容量或最优运行，应后续接入 Pyomo、CVXPY 或其他优化器。

## 后续建议

拿到企业数据后，优先补齐以下几类输入：

- 登陆点逐时接纳能力或典型日接纳曲线。
- 海缆、光缆、管道距离和工程报价。
- 电价、算力服务价格、氢气交付价格和合同边界。
- 数据中心 PUE、IT 容量、可迁移任务比例、光缆带宽和可用率。
- 电解槽 SEC、储氢容量、管道/船运能力、运输损耗和变动成本。

这些数据补齐后，模型可以从“参数化子模型”进一步升级为 8760 小时运行测算、敏感性分析和投资筛选模型。
