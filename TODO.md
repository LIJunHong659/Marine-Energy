## Day 3：整理符号、约束和基础公式

目标：形成可以写进第四章的小节雏形。

### 统一符号

- [ ] `P_grid,t`：t 时刻通过海缆外送的电功率，MW。
- [ ] `P_grid_del,t`：t 时刻陆上实际接收电功率，MW。
- [ ] `P_compute,t`：t 时刻算力中心总用电功率，MW。
- [ ] `P_it,t`：t 时刻 IT 设备用电功率，MW。
- [ ] `B_fiber,t`：t 时刻光缆占用带宽，Gbps 或 Tbps。
- [ ] `P_h2_el,t`：t 时刻电解制氢用电功率，MW。
- [ ] `H_prod,t`：t 时刻制氢量，kg/h。
- [ ] `H_pipe,t`：t 时刻管道输氢量，kg/h。
- [ ] `H_ship,t`：t 时刻等效船运输氢量，kg/h。
- [ ] `H_storage,t`：t 时刻储氢量，kg。
- [ ] `P_marine,t`：t 时刻海洋综合用能功率，MW。

### 电力-海缆基础约束

- [ ] 外送容量约束：

```text
0 <= P_grid,t <= min(P_grid_max, P_grid_accept_max)
```

- [ ] 陆上接收功率：

```text
P_grid_del,t = P_grid,t * (1 - loss_cable)
```

- [ ] 电力外送收益：

```text
R_power,t = P_grid_del,t * price_power
```

### 算力-光缆基础约束

- [ ] 算力中心功率约束：

```text
P_compute_min <= P_compute,t <= P_compute_max
```

- [ ] PUE 关系：

```text
P_compute,t = P_it,t * PUE
```

- [ ] 光缆带宽约束：

```text
0 <= B_fiber,t <= B_fiber_max
```

- [ ] 算力收益可以先用简化表达：

```text
R_compute,t = P_it,t * price_compute
```

备注：如果后续拿不到算力价格，可先用“单位 IT 电耗收益”近似；如果拿到 GPU 小时价格，再换成 GPU 小时模型。

### 氢气-管道 / 船运基础约束

- [ ] 制氢量：

```text
H_prod,t = P_h2_el,t * eta_el
```

- [ ] 输氢分配：

```text
H_prod,t = H_pipe,t + H_ship,t + ΔH_storage,t
```

- [ ] 管道能力约束：

```text
0 <= H_pipe,t <= H_pipe_max
```

- [ ] 船运等效能力约束：

```text
0 <= H_ship,t <= H_ship_max
```

- [ ] 储氢容量约束：

```text
0 <= H_storage,t <= H_storage_max
```

- [ ] 氢气收益：

```text
R_h2,t = (H_pipe,t + H_ship,t) * price_h2
```

### 接入总功率平衡

你负责的模型需要接到总功率平衡里：

```text
P_available,t
=
P_grid,t
+ P_compute,t
+ P_h2_el,t
+ P_marine,t
+ P_storage_ch,t
+ P_curt,t
```

其中你主要负责：

```text
P_grid,t + P_compute,t + P_h2_el,t + P_marine,t
```

这四项的工程含义、容量约束、收益和成本。
